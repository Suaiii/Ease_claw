## 一、2:30 标准版（主稿，目标 2:20-2:45 落地）

### 0:00 – 0:15 　开场 · 指屏定位
**操作**：Chrome 已开 `http://127.0.0.1:8765`，顶栏 `Runtime=OpenClaw`、`Model=gemini-2.5-flash`、`ADB=Connected`（绿点），右侧 contacts 列出 `10086 / 12306 / Mom`。
> "**ClawEase 是 OpenClaw 在视障和老人无障碍场景的落地**。让他们用一句中文完成打电话、发短信、识别诈骗。
> 屏幕顶上这一栏就是 OpenClaw 的意图解析层，我们所有语音都要过它。"

### 0:15 – 0:45 　一击 · 打电话（DIAL 授权卖点）
**操作**：点麦克风按钮，说 `给 10086 打电话` → textarea 自动填入 → 点 Run。
- 屏幕上 `Execution Trace` 显示 `intent = {action:"call", target:"10086"}`，耗时 1-2s
- 手机拨号器弹出 10086 预填号码，**不自动拨出**

> "我说一句中文，OpenClaw 解析出 `action=call`、`target=10086`。
> **注意手机弹的是拨号器不是直接拨出** —— DIAL 不 CALL。老人按绿键 = 每次代操作都有一次物理授权日志。
> 这样既不撞 Android 10+ 权限红线，也保留了老人最后的决定权。"

### 0:45 – 1:30 　杀招 · 识别验证码 + 生活信息归类
**操作**：语音 `有没有新短信` → Run → inbox 区渲染 3 张卡片：
- 验证码 pill（杀招）"不要告诉任何人"
- 话费订单 pill（运营商业务通知，中性标签）
- 正常 pill（家人消息，无前缀）

> "OpenClaw 做 zero-shot 五类分类：冒充银行 / 冒充公检法 / 话费订单 / 快递通知 / 验证码。
> 注意这不只是反诈 —— **话费订单**这一类是合法的运营商业务通知，AI 替老人做『哪条重要、哪条可以略过、哪条要警惕』的归类。
> 真正的杀招是这条**验证码**。朗读时在前面加一句『不要告诉任何人』。
> 为什么？**验证码是中国老人社工诈骗的头号入口** —— 骗子让老人复述验证码，就完成了转账授权。
> 我们不删不屏蔽，只做朗读时的警告标注 —— 原始信息老人永远能访问，我们只在他耳朵边多说一句『这是验证码，别告诉任何人』。"

### 1:30 – 2:00 　边界 · 发短信停在编辑器（伦理硬约束）
**操作**：语音 `给 12306 发一条我晚点到` → Run → 手机 SMS app 打开 compose，收件人 12306，正文"我晚点到"已预填，**光标停在发送键前**。

> "短信默认**不自动发**。我们做了一条伦理硬约束：所有代操作必须有用户最终确认。
> 老人自己按发送键。代理的职责止步于『把话写好、摆到他面前』。"

### 2:00 – 2:20 　叙事收口 · OpenClaw 的位置
**操作**：用鼠标指顶栏 Runtime 行。

> "整套 demo 里，OpenClaw 做了三件事：语音意图解析、反诈分类、未来还要做多轮联系人澄清和工具调用。
> 下面 3 个 ActionAdapter 按 Android Intent 和 ContentResolver 执行，**IM 平台中立** —— 明天换微信、WhatsApp、Line 都是同一套 seam。
> 这是 OpenClaw 落地的典型形态：**智能编排层 + 平台中立执行层**。"

### 2:20 – 2:30 　收尾 · 路线图
> "Layer 2 我们要做方言 ASR 和多轮澄清，Layer 3 打包成 Kotlin 原生 APK。
> 今天这一套，已经在我手边这台小米 13 上跑通。"

> "为什么分三层？Layer 1 今天这套是**无障碍代理的最小可用骨架** —— 一个老人只要会说『打电话』、『发短信』、『有新短信吗』，他就能独立使用一部智能手机。

> 多轮、工具调用、agent mode 都是它已经内置的能力，ClawEase 只是把它接上了 Android 真机。

## 六、评委可能锐利问题（各备 30 字答案）

- **"这不就是把 Google Assistant / Siri 包了一层吗？"**
  "那些是闭源云服务，隐私、方言、反诈专项都无法定制。ClawEase 是开源编排层 + 可审计的本地动作日志。"

- **"反诈分类准确率多少？"**
  "zero-shot 场景 OpenClaw 大模型 + 本地正则双路。准确率我们还在攒测试集；但即便误判也只是加一句朗读警告，不删不屏蔽，错误成本很低。"

- **"话费订单这类合法消息为什么也要单独分？"**
  "老人最关心两件事 —— 别被骗、别漏缴。话费扣款提醒属于『该听到但不该被警告吓到』，中性标签让 AI 能替老人做信息优先级筛选，不只是反诈过滤器。"

- **"老人真的会说『给 10086 打电话』这么规范吗？"**
  "会说『打电话给移动』或者『10086』就够了 —— 正则兜底 + OpenClaw 同义词归一都已经处理。Layer 2 会加方言。"

- **"隐私？短信内容要上传给 LLM 吗？"**
  "只发必要片段（正文）过 OpenClaw 分类，不传联系人全表。Layer 3 原生 APK 会走端侧小模型。"

- **"为什么不做微信代理 / 直接接微信？"**
  "试过，微信风控会踢下线，且灰产风险大。我们选 Android 系统级 intent 路径 —— 平台中立，未来接 WhatsApp、Line 一视同仁。"

- **"和市面上『大字版』『老人模式』相比？"**
  "大字版改的是视觉，ClawEase 改的是交互模态 —— 视障老人看不见大字。我们是**口说即操作**。"
